# 极验验证接入说明

## 已完成的功能

1. **前端极验验证组件** (`src/components/misc/GeetestCaptcha.svelte`)
   - 集成了极验GT4验证码
   - 支持滑动验证
   - 验证结果24小时内有效（存储在localStorage）
   - 响应式设计和主题适配

2. **带验证的评论组件** (`src/components/misc/GiscusWithCaptcha.astro`)
   - 在Giscus评论前添加极验验证
   - 验证通过后才加载评论框
   - 保持原有Giscus功能不变

3. **页面更新**
   - 文章页面 (`src/pages/posts/[...slug].astro`) - 已更新
   - 友链页面 (`src/pages/friends.astro`) - 已更新

4. **后端验证示例** (`geetest-verify.js`)
   - Node.js Express服务器示例
   - 提供完整的极验token验证
   - 可部署到Vercel、Netlify等平台

## 使用方式

### 前端验证（已实现）
1. 用户访问博客文章或友链页面
2. 看到极验验证组件
3. 完成滑动验证
4. 验证通过后显示Giscus评论框
5. 24小时内再次访问无需重复验证

### 后端验证（可选）
1. 部署 `geetest-verify.js` 到服务器
2. 取消前端组件中的注释（`verifyWithBackend`函数）
3. 设置正确的API地址

## 部署后端验证

### 本地测试
```bash
node geetest-verify.js
```

### Vercel部署
1. 创建 `api/verify-geetest.js` 文件
2. 复制 `geetest-verify.js` 中的代码
3. 修改为Vercel函数格式

### 验证端点
- `POST /api/verify-geetest` - 验证极验token
- `GET /api/health` - 健康检查

## 自定义配置

### 修改验证类型
在 `GeetestCaptcha.svelte` 中修改 `riskType` 参数：
- `slide` - 滑动验证（默认）
- `icon` - 图标点选
- `word` - 文字点选
- `ai` - 一点即过

### 修改有效期
修改 `checkPreviousVerification` 函数中的时间判断（24小时）

### 样式自定义
修改组件中的CSS样式类

## 注意事项

1. **极验脚本加载**：组件会自动加载极验SDK
2. **网络错误处理**：网络异常时仍允许显示评论
3. **本地存储**：使用localStorage存储验证状态
4. **主题适配**：自动跟随博客主题切换

## 故障排除

### 验证码不显示
1. 检查网络连接
2. 查看浏览器控制台错误
3. 确认极验ID是否正确

### 验证失败
1. 检查极验控制台配置
2. 确认域名在白名单中
3. 测试不同的验证类型

### 评论框不显示
1. 确认验证是否成功
2. 检查localStorage中是否有验证状态
3. 查看Giscus配置是否正确
